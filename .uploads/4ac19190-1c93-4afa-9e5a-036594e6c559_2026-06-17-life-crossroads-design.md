# 人生十字路口 · 决策教练

> TRAE AI 创造力大赛 · 设计规格文档
> 日期: 2026-06-17

---

## 1. 产品概述

### 1.1 一句话定位

**AI 决策教练**——不是替你做决定，而是 summon 多位专家顾问，帮你理清纠结、看见盲区、找到最适合你的路。

### 1.2 核心价值

- **多视角分析**：不是单一 AI 的意见，而是由多个专业视角构成的立体画像
- **深度追问**：像真正的教练一样，通过对话挖掘用户没说出来的真实偏好
- **可执行的结论**：不止输出分析，还给出分阶段行动路径

### 1.3 目标用户

- **主人群**：18-35 岁，面临职业选择、城市变迁、教育路径、人生重大决策的年轻人
- **泛人群**：任何需要做复杂决策的场景，包括感情、消费、生活方式选择

## 2. 与 Professor Synapse 的设计融合

| Synapse 能力 | 我们的应用 |
|---|---|
| **Agent Summoning** | 根据决策类型，动态召唤三位专家组成顾问团（如：职业顾问 + 生活教练 + 财务规划师） |
| **Context Gathering** | 每位专家各自向用户提问，从不同维度收集信息 |
| **Convener Protocol** | 教练主持多专家圆桌辩论，不同视角产生碰撞再合成 |
| **Memory System** | 记录用户的价值观偏好和决策模式，跨 session 复用 |

## 3. 核心交互流程（四阶段）

### Phase 1 · 决策识别

用户说出自己的纠结，AI 判断决策类型。

- 输入：用户的自然语言困惑
- 输出：识别出的决策类别（职业 / 生活 / 感情 / 消费 / 综合）
- 行为：教练对用户说"我为你 summon 以下专家"，展示三位专家身份

**场景示例**

> 用户："我纠结要不要从上海回成都"
> 教练："这是一个职业 x 城市 x 生活方式的多维决策。我为你 summon 三位专家：职业发展顾问、生活方式教练、财务规划师"

### Phase 2 · 各专家独立分析

每位专家依次向用户提问，从自己的专业视角收集信息。

- 每位专家提出 2-3 个针对性问题
- 用户回答后，该专家先做初步判断沉淀
- 等所有专家完成访谈后，进入下一阶段

**默认专家库（根据决策类型动态调整）**

| 专家角色 | 擅长领域 | 追问方向 |
|---|---|---|
| 职业发展顾问 | 职业路径、行业趋势、技能成长 | 职位发展空间、行业前景、竞争力 |
| 生活方式教练 | 幸福感、压力、社交、日常节奏 | 生活满意度、社交需求、倦怠程度 |
| 财务规划师 | 收入、储蓄、房价、长期财务 | 实际储蓄率、买房可行性、财务安全感 |
| 情感关系教练 | 亲密关系、家庭关系、社交圈 | 关系质量、家人期望、社交支持系统 |
| 教育规划师 | 学习路径、技能投资、学历价值 | 学习目标、投入产出比、长期成长 |

### Phase 3 · 专家辩论

教练主持圆桌辩论，三位专家基于同一组用户数据给出各自的结论和建议。

- 专家之间可以不同意见——这是产品核心价值
- 教练负责总结共识、点明分歧、追问决策者（用户）的倾向
- 用户可以选择"我现在倾向于 X，你们怎么看"

### Phase 4 · 综合报告

输出一份结构化的决策报告卡，包含四部分：

1. **专家加权对比图** — 每位专家对各个选项的打分，可视化展示
2. **核心矛盾点** — AI 从对话中提炼用户真正的纠结和价值观冲突
3. **分阶段策略建议** — 不是"选 A 还是 B"，而是"如果选 A，接下来三步是什么"
4. **下一步行动清单** — 具体可执行的步骤

## 4. 技术架构

### 4.1 整体设计

`
[前端] Vue 3 SPA
    | 用户输入 / 对话流
[后端] PHP API 服务
    | Agent 调度
[AI Agent 层] 多专家决策引擎
    |-- 决策识别器 (Decision Classifier)
    |-- 专家角色管理器 (Expert Role Manager)
    |-- 辩论主持人 (Debate Moderator)
    |-- 报告生成器 (Report Generator)
`

### 4.2 AI Agent 层核心组件

**决策识别器**
- 输入：用户的初始自然语言输入
- 输出：决策类别标签 + 推荐专家阵容
- 技术：通过 LLM prompt engineering 实现场景分类

**专家角色管理器**
- 维护专家角色库（可以是 prompt 模板 + 角色定义）
- 根据决策类别自动选择 3 位专家
- 每位专家有独立的 persona prompt、追问策略

**辩论主持人**
- 管理多轮对话的上下文
- 跟踪每位专家的信息收集进度
- 主持辩论：决策教练角色
- 生成共识和分歧摘要

**报告生成器**
- 将对话历史和专家结论结构化
- 生成可视化数据（估值打分 + 权重计算）
- 输出自然语言的综合分析

### 4.3 数据流

`
用户输入 -> 决策识别 -> 选择专家团
    |
    v
专家 1 提问 -> 用户回答 -> 专家 1 沉淀
专家 2 提问 -> 用户回答 -> 专家 2 沉淀
专家 3 提问 -> 用户回答 -> 专家 3 沉淀
    |
    v
教练主持辩论 -> 用户反馈倾向
    |
    v
综合报告生成 -> 展示给用户
    |
    v
(可选) 记忆存储
`

### 4.4 技术选型

| 层级 | 技术 | 说明 |
|---|---|---|
| 前端框架 | Vue 3 | 用户已掌握 |
| 后端 | PHP | 用户已掌握，处理 API 路由、session 管理等 |
| AI 引擎 | LLM API (如 Claude / GPT) | 通过 PHP 后端调用 |
| 对话存储 | 轻量 (如 SQLite / JSON) | 记录对话历史 + 决策日志 |
| 可视化 | ECharts / Chart.js | 决策报告卡中的对比图 |

## 5. MVP 范围（初赛）

初赛需要交付一个能跑的 Demo。建议 MVP 功能：

- [ ] 用户输入场景 -> AI 识别决策类型 + 召唤 3 个默认专家
- [ ] 三位专家依次提问（每人 1-2 轮）
- [ ] 教练主持简单辩论/总结
- [ ] 生成结构化的决策报告卡（纯文本 + 简单可视化）
- [ ] 基本的对话 UI（聊天界面形式）

**不包含（复赛再扩展）：**
- 记忆系统
- 专家角色自定义
- 多语言
- 移动端适配

## 6. 竞品差异点

| 维度 | 一般 AI 决策工具 | 我们 |
|---|---|---|
| 视角 | 单一 AI 视角 | 多专家多视角 |
| 深度 | 一次问答出结论 | 多轮追问挖掘隐性偏好 |
| 输出形式 | 直接给建议 | 先给框架，用户自己填空 |
| 记忆 | 无 | 跨 session 记录价值观 |
| 争议处理 | 回避矛盾 | 主动呈现分歧 |

---

## 7. 未知与风险

- **LLM 成本**：多轮多专家对话 token 消耗较大，需要设计 prompt 优化
- **"专家幻觉"**：专家角色可能给出不准确的分析，需要后处理校验
- **用户耐心**：多轮追问可能导致用户中途放弃，需要设计渐进式交互
- **决策类型覆盖**：初期难以覆盖所有决策类型，建议先验证"职业 + 城市"场景

---

## 8. 成功指标（比赛评审视角）

- **体验完整性**：用户是否能完整走完 4 个阶段并拿到有用结论
- **AI 能力展示**：多专家辩论的差异化和合成是否自然
- **情感共鸣**：用户是否觉得"这个 AI 真的理解我"
- **技术实现**：Agent 调度、上下文管理是否稳定
